Các kiểu dữ liệu cơ bản trong Java

by Chau - 17 Tháng 10 6521 Lượt xem 0 Bình luận

Trong lập trình Java, việc hiểu rõ các kiểu dữ liệu cơ bản là bước nền tảng quan trọng giúp lập trình viên sử dụng bộ nhớ hiệu quả, tránh lỗi và đảm bảo chương trình chạy ổn định. Java là ngôn ngữ lập trình mạnh kiểu (strongly typed), nghĩa là mọi biến đều phải có kiểu dữ liệu xác định ngay từ khi khai báo. Kiểu dữ liệu quyết định dung lượng bộ nhớ mà biến chiếm, giá trị hợp lệ mà nó có thể chứa, và các phép toán có thể thực hiện.

Java chia dữ liệu thành hai nhóm lớn: kiểu nguyên thủy (Primitive types) và kiểu tham chiếu (Reference types). Mỗi nhóm có vai trò khác nhau trong việc lưu trữ và xử lý thông tin.

1. Kiểu nguyên thủy (Primitive Types)

Kiểu nguyên thủy là những kiểu dữ liệu có sẵn trong Java, được lưu trực tiếp trong bộ nhớ Stack, giúp chương trình truy xuất nhanh. Có tổng cộng 8 kiểu dữ liệu nguyên thủy:

  • byte (8-bit): dùng lưu số nhỏ, thường cho mảng lớn để tiết kiệm bộ nhớ. Phạm vi: -128 đến 127.
  • short (16-bit): dùng cho số nguyên vừa phải. Phạm vi: -32,768 đến 32,767.
  • int (32-bit): kiểu nguyên mặc định trong Java. Dùng hầu hết các phép toán số học.
  • long (64-bit): dùng cho giá trị nguyên rất lớn. Khi khai báo cần hậu tố L, ví dụ long n = 5000000L;
  • float (32-bit): lưu số thực, độ chính xác đơn. Cần hậu tố F khi khai báo.
  • double (64-bit): số thực độ chính xác kép, là kiểu mặc định cho số thực.
  • char: lưu một ký tự Unicode đơn (ví dụ 'A', '1', '\n').
  • boolean: chỉ chứa hai giá trị true hoặc false, thường dùng trong điều kiện rẽ nhánh.

// Ví dụ khai báo các kiểu dữ liệu cơ bản
int age = 21;
long population = 8000000000L;
float ratio = 0.75F;
double pi = 3.14159265359;
char grade = 'A';
boolean isPassed = true;

2. Kiểu tham chiếu (Reference Types)

Khác với kiểu nguyên thủy, kiểu tham chiếu lưu địa chỉ vùng nhớ của đối tượng (object) trong Heap. Bao gồm: String, Array (mảng), Class, Interface và Wrapper Class (ví dụ: Integer, Double). Khi làm việc với kiểu tham chiếu, ta thao tác trên đối tượng thật sự chứ không phải giá trị trực tiếp. 


// Ví dụ về kiểu tham chiếu
String name = "Java";
int[] scores = {85, 90, 95};
Integer number = 100; // Wrapper class - autoboxing
int raw = number;     // Unboxing từ Integer sang int

Khi khai báo một biến kiểu tham chiếu, bản thân biến không lưu dữ liệu thực tế mà chỉ trỏ đến đối tượng chứa dữ liệu. Vì thế, nếu hai biến cùng tham chiếu đến một đối tượng thì việc thay đổi một biến sẽ ảnh hưởng đến biến còn lại.

3. Ép kiểu (Casting) trong Java

Ép kiểu cho phép chuyển đổi giữa các kiểu dữ liệu khác nhau. Có hai loại:

  • Widening Casting (tự động): khi chuyển từ kiểu nhỏ sang kiểu lớn, ví dụ int → long → float → double.
  • Narrowing Casting (thủ công): khi chuyển từ kiểu lớn sang kiểu nhỏ, có thể mất dữ liệu. Cần dùng dấu ngoặc tròn, ví dụ (int) 3.14.

// Ví dụ ép kiểu
double a = 9.78;
int b = (int) a;   // Narrowing - b = 9
int x = 10;
double y = x;      // Widening - tự động

Khi ép kiểu, lập trình viên cần cẩn trọng để không làm mất dữ liệu hoặc gây lỗi tràn số (overflow). Với kiểu char và int, Java cho phép chuyển đổi trực tiếp để xử lý ký tự ASCII, ví dụ (int)'A' cho ra 65.

4. Từ khóa var trong Java 10+

Từ phiên bản Java 10, có thể dùng var để suy luận kiểu dữ liệu cho biến cục bộ. Tuy nhiên, Java vẫn giữ tính tĩnh — nghĩa là kiểu được xác định tại thời điểm biên dịch, không thể thay đổi sau đó. 


var message = "Hello Java";  // Kiểu String
var number = 42;             // Kiểu int

5. Một số lưu ý quan trọng

  • So sánh chuỗi dùng equals() thay vì ==.
  • Tránh chia cho 0 với kiểu số nguyên; nếu chia 0.0 sẽ ra Infinity với kiểu double.
  • Cẩn thận khi ép kiểu long → int có thể gây tràn dữ liệu.
  • Đối với tiền tệ hoặc số có độ chính xác cao, nên dùng BigDecimal.
  • Kiểu boolean không thể ép trực tiếp sang số như trong C/C++.

6. Kết luận

Nắm vững các kiểu dữ liệu trong Java là bước đầu tiên để viết chương trình hiệu quả và dễ bảo trì. Khi hiểu cách hoạt động của từng kiểu, lập trình viên có thể lựa chọn đúng loại cho từng trường hợp: tiết kiệm bộ nhớ, tăng hiệu suất và giảm lỗi. Sau khi thành thạo phần này, bạn nên tìm hiểu sâu hơn về kiểu dữ liệu phức tạp như ArrayList, HashMap, hoặc cơ chế Generics trong Java.
